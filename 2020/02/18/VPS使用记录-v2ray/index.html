VPS使用记录-v2ray

February 18, 2020 This post has been read by person

V2ray的安装推荐脚本：

bash <(curl -s -L https://git.io/v2ray.sh)

此脚本支持TCP，HTTP/2，WebSocket，mKCP等多个传输协议，如果是在一部纯净的VPS上可直接使用脚本按提示配置即可。

然而这原本的机器上已经使用宝塔安装了LNMP，与脚本安装WebSocket+TLS使用nginx配置TLS有冲突，故而在安装带有TLS机制（例如WebSocket+TLS和HTTP/2）的传输协议时，不能使用脚本自动配置，而应该手动操作。

需要指出的是，受限于IOS客户端我购买的是QUANTUMULT，传输协议只支持WS和HTTP，因而实际上最终可选传输协议不多，在测试过程中，脚本的TCP_HTTP协议在IOS和WINDOWS端都可用，但是速度太慢，且没有TLS担心被Q（SSR的前车之鉴……），mKCP使用UDP无连接和快速重传机制，在速度上有了质的飞跃，可惜QUANTUMULT不支持，否则是一个很好的选择，安装也简便，不用考虑TLS配置。

执行命令yum install -y wget && wget -O install.sh http://www.aapanel.com/script/install_6.0_en.sh && bash install.sh，安装完宝塔和部署好网站后，准备一个已设置好解析到该VPS的域名，添加一个新的网站并绑定该域名，并开启SSL，接着在VPS执行v2ray安装脚本，按提示输入，选择不自动配置TLS，其余等均默认即可，选择WebSocket+TLS传输协议安装完成后，在v2ray的配置信息中需要注意的有地址，用户ID，额外ID和路径。

接着，在宝塔面板v2ray地址（域名）所绑定的网站里的配置文件中，在#SSL-END下加入

保存即可，至此，服务器端WebSocket+TLS安装完成。

至于CDN，由于cloudflare套上去会大大拖慢速度，国内的我不敢用，故而除非IP被Q，否则不推荐，有TLS做保障个人使用应该问题不大。

Windows客户端使用的是v2rayN，这只是一个UI壳子，方便使用，地址在github上找，另外还需要v2ray-core，这才是真正的项目文件，下载地址 （客户端和服务器是一样的，如果服务器（Linux）不是使用一键安装脚本，可以使用Project V的方法bash <(curl -L -s https://install.direct/go.sh) 安装v2ray，接着手动修改/etc/v2ray/config.json文件配置。），然后将v2ray-core文件放到v2rayN的文件夹中，运行v2rayN.exe，然后添加VMess服务器，按提示输入相关参数即可，底层传输安全记得选TLS。至于PAC规则，使用默认或自行搜索相关策略都行。

IOS客户端使用的是QUANTUMULT（国区已下架），设置里添加VMess服务器，打开传输和TLS，其中传输的类型有两个，http即一键安装脚本中的TCP_HTTP，ws则是WebSocket+TLS所需要的，TLS里的验证选0，其余按提示填写或默认即可。

使用后体验，稳定性比SSR要好很多，但是WebSocket+TLS新建连接时所需要的握手时间有点长，刚开始会有断线感觉的不适应，这一点不如mKCP，速度上也稍逊一筹。
