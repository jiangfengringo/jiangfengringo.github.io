VPS使用记录-Virtual Private Network

January 17, 2020 This post has been read by person

18年之后，lantern和psiphon基本上就处于瘫痪状态，偶尔回光返照，恰好听朋友推荐搬瓦工，所以花点时间鼓捣了一下。

一开始是买了个KVM，配置丐版，服务三无，但是遭不住便宜，找了个L2TP/IPSEC配置脚本直接就上手了，

wget –no-check-certificate https://raw.githubusercontent.com/teddysun/across/master/l2tp.sh

使用后的体验就像，下午八点半的移动宽带，绿皮火车哐叱哐叱，而且时常掉线，当时还考虑是美机不稳定，换成港机也并无好转，查了些信息才发现，14年GWF就针对L2TP做了特别照顾，没被Q估计还得感谢为验证机器正常运行而随手发布的apache页面，亦或者是连接时间不长懒得管。

后续了解了一些与之相关的一些协议，做了一下梳理：

加密协议 优势 缺点 备注
PPTP（不推荐） 全平台，易于安装，计算量小速度快 非常不安全，已被NSA破解，易于封锁 本身只是通讯协议，数据安全性由其他验证方式提供，要求TCP端口1723和GRE协议
L2TP（不苛求数据绝对安全的情况下可用，也推荐那些不支持OpenVPN的传统设备） 通常被视为安全，易于安装，所有现代平台可用，比OpenVPN更快，能对抗严格防火墙 可能已被NSA破解，似乎被NSA刻意削弱，经常执行错误 协议本身不提供加密，而是使用IPsec验证套件协助，且只能用部分端口，所以面对NAT防火墙会容易被封锁，L2TP/IPsec压缩数据两次，但是用内核加解密且支持多线程，所以理论上快于OpenVPN，PSKs只用来授权连接，加密则用其他例如AES
SSTP（微软垄断，不推荐使用） 非常安全，windows完全整合，微软支持，能越过大多数防火墙 微软完全拥有专利标准 使用SSL3.0的加密方式，可以使用TCP端口443，在windows平台上最稳定
IKEv2（针对移动设备比OpneVPN更合适） 快，稳定，特别当网络切换或重连时，安全（只要使用AES），容易安装（至少在用户端），支持黑莓设备 许多平台不支持，服务器端安装很麻烦，容易导致一些潜在的问题，只有开源实现才是可信的 早期由微软和思科共同开发，支持windows7+，黑莓，IOS设备，但要小心微软开发的东西，所以只有支持Linux和其他操作系统的独立开发兼容版本的IKEv2才是值得信任的，IKEv2时IPsec协议套件的一部分，通过处理SA来保证流量安全，适用于移动设备。比L2TP更优秀。
OpenVPN（推荐使用，综合考量下最好） 非常安全（只要使用PFS），高度可配置，开源，能越过防火墙 需要第三方软件 使用OPENSSL库和TLS协议，可在任意端口运行，包括TCP端口443，最好是在UDP端口运行，可以使用OPENSSL库提供加密支持，OpenVPN加密包含两部分，数据频道加密和控制频道加密，保护数据安全和连接安全。
